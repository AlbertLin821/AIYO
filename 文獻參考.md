## 2. 旅遊推薦與影片分段相關研究

### 2.1 多模態旅遊推薦

- **研究主題**：利用視覺特徵與文字描述來推薦旅遊路線與景點
- **關鍵技術**：
  - 圖片與文字的多模態融合
  - 景點推薦演算法
  - 路線規劃優化
- **應用場景**：旅遊推薦系統、景點識別、路線規劃

### 2.2 影片語意分段與摘要

#### Google Research: Semantic Video Trailers

- **研究內容**：根據語意將影片切成片段並生成 trailer
- **關鍵技術**：
  - Semantic segmentation
  - 語意相似度計算
  - 片段摘要生成
- **應用**：影片自動分段、精華片段提取

#### Semantic segmentation of documentary video using music breaks

- **研究內容**：利用音樂斷點做視聽分段
- **關鍵技術**：
  - 音訊特徵提取
  - 音樂段落識別
  - 視聽同步分段
- **應用**：紀錄片分段、影片結構分析

#### 深度學習語意影像/影片分割綜述

- **研究內容**：深度學習在語意分割的應用綜述
- **關鍵技術**：
  - CNN / Transformer 模型
  - 語意分割網路架構
  - 多尺度特徵融合
- **應用**：影片理解、場景分割

## 3. API 官方文件

### 3.1 YouTube API

#### YouTube Data API v3

- **用途**：搜尋旅遊影片、取得影片 metadata
- **主要端點**：
  - `search.list`：搜尋影片
  - `videos.list`：取得影片詳情
  - `channels.list`：取得頻道資訊
- **配額限制**：每日 10,000 units（預設）
- **文件連結**：https://developers.google.com/youtube/v3

#### YouTube Captions API

- **用途**：取得影片字幕
- **主要端點**：
  - `captions.list`：列出可用字幕
  - `captions.download`：下載字幕（SRT/VTT）
- **文件連結**：https://developers.google.com/youtube/v3/docs/captions

### 3.2 Google Maps API

#### Google Maps Places API

- **用途**：取得景點地理資訊
- **主要功能**：
  - 地點搜尋
  - 地點詳情
  - 地點照片
- **文件連結**：https://developers.google.com/maps/documentation/places/web-service

#### Google Maps Directions API

- **用途**：取得路線規劃
- **主要功能**：
  - 路線計算
  - 多點路線
  - 交通方式選擇
- **文件連結**：https://developers.google.com/maps/documentation/directions

#### Google Maps Distance Matrix API

- **用途**：計算多點間距離與時間
- **主要功能**：
  - 批次距離計算
  - 多種交通方式
  - 即時交通狀況
- **文件連結**：https://developers.google.com/maps/documentation/distance-matrix

  - 其他開源模型
- **文件連結**：https://ollama.com/library

### 4.2 開源 LLM 比較與推薦

#### Qwen 模型

- **特色**：中文能力極佳
- **適用場景**：中文對話、中文理解任務
- **參考資料**：多個 blog 指出其在中文任務與指令跟隨上與 GPT-4o-mini 接近

#### Llama 3.3 模型

- **特色**：全能型，多項 benchmark 表現優異
- **適用場景**：通用對話、工具呼叫
- **參考資料**：MMLU, IFEval 等 benchmark 接近或略高於 GPT-4o-mini

#### Mistral 模型

- **特色**：高 throughput、低延遲
- **適用場景**：高併發、低延遲需求
- **參考資料**：效能測試與比較文章

### 4.3 GPT-4o-mini vs 開源模型比較

#### Benchmark 比較

- **MMLU**：多任務語言理解
- **IFEval**：指令跟隨評估
- **中文任務**：中文對話、理解能力
- **價格比較**：開源模型成本為 0

#### 參考資料

- 多個技術 blog 與 benchmark 報告
- 模型效能測試結果
- 實際應用案例

### 4.4 vLLM vs Ollama 性能與併發比較

#### vLLM 特色

- **高吞吐量**：Continuous batching
- **低延遲**：PagedAttention 技術
- **生產就緒**：適合大量併發

#### Ollama 特色

- **易於使用**：簡單的部署與管理
- **開發友好**：適合開發與測試
- **資源需求低**：較低的硬體需求

#### 使用建議

- **開發階段**：使用 Ollama
- **生產環境**：使用 vLLM（大量併發時）

### 4.5 Kubernetes 上部署 Ollama / vLLM

#### 部署教學

- **Kubernetes 設定**：Deployment, Service 配置
- **GPU 資源管理**：Node selector, Resource limits
- **自動擴展**：HPA（Horizontal Pod Autoscaler）設定
- **監控與日誌**：Prometheus, Grafana 整合
