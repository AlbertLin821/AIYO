## 三、功能不足與缺陷

### 3.1 側欄登入狀態不一致（高）

**現象**: 已登入狀態下，在 **Explore、Saved、Trips、Updates、Create、Settings** 各頁的側欄仍顯示「Log in」「Sign up」，而非使用者頭像與選單（Settings / Log out）。

**原因**: 僅主頁 `app/page.tsx` 有呼叫 `/api/auth/me` 並將 `user` 傳入 `<AppSidebar>`；其餘頁面使用 `<AppSidebar />` 時未傳入 `user`，因此側欄一律顯示未登入狀態。

**影響頁面**:  
`app/explore/page.tsx`、`app/saved/page.tsx`、`app/trips/page.tsx`、`app/updates/page.tsx`、`app/create/page.tsx`、`app/settings/page.tsx`。

**建議**:  
- 在共用 layout 或各頁面取得目前使用者（例如呼叫 `/api/auth/me` 或共用 context），並將 `user`、必要時連同 `chatCount`、`savedCount`、`onNewChat` 傳入 `<AppSidebar>`；或  
- 改用已支援 `user` 的 `MainLayout` 於上述頁面，以統一登入狀態顯示。

### 3.2 Google 地圖無法載入（高）

**現象**: 主頁地圖區顯示「這個網頁並未正確載入 Google 地圖。請查看 JavaScript 控制台，瞭解相關的技術詳情。」

**Console**:  
`RefererNotAllowedMapError` — 目前網站網址未在 Google Cloud Console 的 API 金鑰「應用程式限制」中列為允許的參照來源。

**建議**:  
在 Google Cloud Console 的該金鑰設定中，將 `http://localhost:3002`（及正式環境網域）加入「HTTP 參照來源」允許清單。  
（註：未修改 .env，依規則不更動環境變數內容。）

### 3.3 表單無障礙（Accessibility）

**Console**:  
`A form field element should have an id or name attribute (count: 1)`  
至少有一個表單欄位缺少 `id` 或 `name`，不利無障礙與表單辨識。

**建議**: 為所有表單欄位加上 `id` 與對應 `<label htmlFor="...">`，必要時加上 `name`。

### 3.4 Google Maps API 載入方式

**Console**:  
`Google Maps JavaScript API has been loaded directly without loading=async.`  
建議改為非同步載入以符合最佳實踐、避免阻塞。

### 3.5 Google Maps 舊版 API 棄用警告

**Console**:  
- `google.maps.DirectionsRenderer`、`google.maps.DirectionsService` 已標示為 deprecated（2026 年 2 月起）。  
官方建議改用 `google.maps.routes.Route.computeRoutes` 等新 API。

**建議**: 規劃遷移至新版 Routes API，以符合日後 Google 政策。

### 3.6 登入頁「Close」按鈕行為

**現象**: 登入／註冊彈窗上的關閉按鈕會執行 `router.push("/home")`，會離開登入頁並進入首頁，而非僅關閉彈窗。若設計為「關閉後仍留在登入頁」，目前行為與一般「關閉」預期可能不符。

**建議**: 確認產品需求：關閉時是「回首頁」還是「僅關閉彈窗留在登入頁」，再調整實作。

## 五、建議修復優先順序

1. **高**: 新增或修正 `/inspiration`、`/terms`、`/privacy`（或改連結），避免 404。  
2. **高**: 側欄登入狀態一致：在 Explore / Saved / Trips / Updates / Create / Settings 傳入 `user` 給 `AppSidebar`（或改用 MainLayout）。  
3. **高**: Google 地圖 Referer 設定，使主頁地圖可正常載入。  
4. **中**: 表單欄位補上 `id`/`name` 與 label 關聯，改善無障礙與 Lighthouse。  
5. **中**: 登入頁關閉按鈕行為與產品預期對齊。  
6. **低**: Google Maps 改為 async 載入、規劃遷移 Directions 至新版 Routes API。
