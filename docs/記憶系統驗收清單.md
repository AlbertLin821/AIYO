# 記憶系統驗收清單

## 功能驗收

- [ ] 使用者於 A 會話輸入偏好（預算、景點偏好、影片偏好）後，B 會話可被引用。
- [ ] 偏好抽取每 3 輪觸發一次，失敗不阻斷主回覆。
- [ ] 長期偏好可透過語意檢索注入 Prompt（`LIMIT 3-5` + threshold）。
- [ ] 前端可手動觸發「AI 檢查偏好」，並更新近期記憶列表。
- [ ] 前端可切換「每次送出訊息後自動巡檢長期偏好」。

## 安全驗收

- [ ] `ai-service` 僅接受內部 token 或白名單 IP 請求。
- [ ] `ai-service` CORS 僅允許 gateway 來源。
- [ ] `user_preferences` 啟用 RLS 並以 `app.user_id` 隔離資料。
- [ ] gateway 採 `access_token + refresh_token`，可透過 refresh 續期 access token。

## 穩定性驗收

- [ ] `ai-service` 不可用時，gateway 可降級為短期記憶回覆，不中斷聊天。
- [ ] Redis 快取生效（`chat:session:{session_id}`，TTL=1h，保留最近 50 則）。
- [ ] fallback 狀態可在前端提示。

## 效能驗收

- [ ] 建立 Prometheus metrics（gateway/ai-service）。
- [ ] 建立 Sentry error tracking（gateway/ai-service）。
- [ ] Locust 10 user 壓測達成 E2E < 2s 目標。
