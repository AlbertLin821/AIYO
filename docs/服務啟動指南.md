# AIYO 服務啟動指南

本文件說明本機開發環境如何啟動 AIYO 所有必要服務。

## 1. 前置需求

- Docker Desktop（需可啟動容器）
- Python 3.10+
- Node.js 18+
- npm
- Ollama（本地 Windows 安裝版，需先安裝並啟動）

## 2. 環境變數

1. 複製 `.env.example` 為 `.env`
2. 依需求檢查以下欄位：
   - `DATABASE_URL`
   - `REDIS_URL`
   - `OLLAMA_BASE_URL`
   - `OLLAMA_MODEL`
   - `AI_SERVICE_URL`
   - `API_PORT`
   - `NEXT_PUBLIC_API_BASE_URL`（前端建議設為 `http://localhost:3001`）
   - `JWT_SECRET`
   - `JWT_EXPIRES_IN`
   - `YOUTUBE_API_KEY`（若要啟用 YouTube 工具）
   - `ENABLE_WEATHER_TOOL`、`ENABLE_YOUTUBE_TOOL`、`ENABLE_TRANSPORT_TOOL`、`ENABLE_TRAVEL_INFO_TOOL`
   - `TOOL_AGENT_MAX_ROUNDS`、`TOOL_AGENT_MAX_CALLS_PER_ROUND`

## 2.1 執行 migration（既有 DB 必做）

若資料庫不是全新建立，請依序執行：

- `scripts/migrations/004_auth_and_users.sql`
- `scripts/migrations/002_itineraries.sql`（若尚未建立 itineraries 系列表）
- `scripts/migrations/005_link_user_to_itinerary.sql`
- `scripts/migrations/006_video_processing_cache.sql`
- `scripts/migrations/007_chat_sessions_external_id.sql`
- `scripts/migrations/009_user_ai_settings_and_location.sql`
- `scripts/migrations/010_admin_audit_and_dev_console.sql`

## 3. 啟動 Ollama（本地 Windows）

Ollama 使用本地 Windows 安裝版，不透過 Docker 啟動。

1. 確認已安裝 Ollama（https://ollama.com/download）
2. 啟動 Ollama 服務（安裝後預設會自動啟動，監聽 `http://localhost:11434`）
3. 下載所需模型：

```bash
ollama pull qwen3:8b
ollama pull nomic-embed-text
```

健康檢查：瀏覽器開啟 `http://localhost:11434` 應顯示 "Ollama is running"

## 4. 啟動基礎服務（Docker）

在專案根目錄執行：

```bash
docker compose up -d
```

此指令會啟動 PostgreSQL、pgAdmin 與 Redis。

健康檢查建議：

- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`

## 5. 啟動 ai-service（FastAPI）

在 `ai-service` 目錄執行：

```bash
python -m venv .venv
.venv\\Scripts\\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

檢查：

- Health: `http://localhost:8000/health`
- Swagger: `http://localhost:8000/docs`

## 6. 啟動 api-gateway（Express）

在 `api-gateway` 目錄執行：

```bash
npm install
npm run dev
```

檢查：

- Health: `http://localhost:3001/health`
- 模型清單: `http://localhost:3001/api/models`

## 7. 啟動 frontend（Next.js）

在 `frontend` 目錄執行：

```bash
npm install
npm run dev
```

預設網址：

- `http://localhost:3000`

## 8. 建議啟動順序

1. 啟動本地 Ollama 並確認模型已下載
2. `docker compose up -d`（PostgreSQL + Redis）
3. `ai-service`
4. `api-gateway`
5. `frontend`

## 9. 開發者後台

開發環境下可使用固定帳密登入開發者後台。

- 前端頁面：`http://localhost:3000/dev/login`
- 帳號：`admin@gmail.com` / 密碼：`adminadmin`
- 登入後可查看所有使用者資料、對話紀錄、偏好、行程，以及完整稽核紀錄（API 請求/回應、AI prompt/response、工具呼叫）。
- 此功能在 `NODE_ENV=production` 下全部停用。

## 10. 常見問題

### Q1: 前端顯示無法連線 chat/models

- 確認 `api-gateway` 有啟動
- 確認前端的 `NEXT_PUBLIC_API_BASE_URL` 指到 `http://localhost:3001`

### Q1-1: 登入失敗或 token 無效

- 確認 `.env` 的 `JWT_SECRET` 有設定且前後一致
- 若改過 `JWT_SECRET`，請重新登入取得新 token

### Q2: chat 可連線但無回覆

- 確認 `OLLAMA_BASE_URL` 正確
- 確認 Ollama 已下載可用模型（如 `qwen3:8b`）

### Q3: chat history 沒有保存

- 若 Redis 未啟動會自動回退成記憶體模式
- 重啟 gateway 後記憶體模式資料會消失，屬預期行為
