# AIYO 服務啟動指南

這份文件已依照目前專案實作更新，內容包含：
- 現在實際使用的模型
- 正確的啟動順序
- V2 相關環境變數
- Docker / 本機服務的最新啟動方式

## 1. 目前模型分工

### 語音輸入
- 語音轉文字不是由 Ollama 模型負責。
- 目前前端使用瀏覽器的 `Web Speech API`：
  - `SpeechRecognition`
  - `webkitSpeechRecognition`
- 也就是說，語音辨識依賴使用者瀏覽器能力，主要以 Chrome 為主。

### 語意解析
- 語音轉成文字後，前端把文字送到 `POST /api/v2/voice/intent`。
- 目前 `ai-service` 內的 `parse_voice_intent_text()` 以規則解析為主，不是用 LLM 做意圖解析。

### 對話 / 推理模型
- 目前 Ollama 對話模型預設是：
  - `gemma4:e4b`
- 相關環境變數：
  - `OLLAMA_MODEL=gemma4:e4b`

### Embedding 模型
- V2 規格鎖定：
  - `nomic-embed-text`
  - 維度 `768`
- 相關環境變數：
  - `OLLAMA_EMBED_MODEL=nomic-embed-text`
  - `V2_EMBED_MODEL_NAME=nomic-embed-text`
  - `V2_EMBED_DIM=768`

## 2. 前置需求

- Docker Desktop
- Python 3.10+
- Node.js 18+
- npm
- Ollama

## 3. 環境變數設定

1. 將 `.env.example` 複製為 `.env`
2. 依需求調整以下欄位：

```env
DATABASE_URL=postgresql://aiyo:aiyo_password@localhost:5432/aiyo_db
REDIS_URL=redis://localhost:6379

OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=gemma4:e4b
OLLAMA_EMBED_MODEL=nomic-embed-text

AI_SERVICE_INTERNAL_TOKEN=replace_with_long_random_internal_token
AI_SERVICE_ALLOWED_IPS=127.0.0.1,::1

AI_SERVICE_URL=http://localhost:8000
API_PORT=3001
NEXT_PUBLIC_API_BASE_URL=http://localhost:3001

JWT_SECRET=replace_with_secure_jwt_secret
JWT_REFRESH_SECRET=replace_with_secure_refresh_secret

V2_EMBED_MODEL_NAME=nomic-embed-text
V2_EMBED_MODEL_VERSION=1
V2_EMBED_DIM=768
V2_RECOMMEND_SYNC_TIMEOUT_SEC=3.5
V2_PLAN_SYNC_TIMEOUT_SEC=5.0
V2_JOB_POLL_AFTER_MS=1200
V2_YOUTUBE_STATS_TTL_HOURS=24
V2_GEOCODE_MAX_RETRIES=3

LEGACY_FRONTEND_PATH=/legacy
NEXT_PUBLIC_PHASE_C_READONLY=false
```

## 4. 啟動 Ollama

Ollama 需要在 Windows 本機啟動，不在 Docker 內。

先確認 Ollama 已安裝並啟動，然後拉取需要的模型：

```bash
ollama pull gemma4:e4b
ollama pull nomic-embed-text
```

確認服務可用：

```bash
curl http://localhost:11434
```

如果回應類似 `Ollama is running`，表示正常。

## 5. 啟動 Docker 基礎服務

目前 `docker-compose.yml` 會啟動：
- PostgreSQL
- Redis
- pgAdmin

啟動指令：

```bash
docker compose up -d
```

如果你的 Windows 沒有把 Docker 加進 PATH，就用完整路徑：

```powershell
& 'C:\Program Files\Docker\Docker\resources\bin\docker.exe' compose up -d
```

啟動後可檢查：
- PostgreSQL: `localhost:5432`
- Redis: `localhost:6379`
- pgAdmin: `http://localhost:5050`

## 6. 套用資料庫 migrations

這一步現在很重要，因為 V2 schema 已經加入專案。

不要只手動記舊 migration 檔名，直接跑整套 migration：

```bash
npm --prefix api-gateway run migrate:deploy
```

這會依序執行 `scripts/migrations/` 內所有 migration，包含：
- 舊有 auth / itinerary / audit schema
- V2 的 `014_v2_core_schema.sql`

## 7. 啟動 ai-service

在 `ai-service` 目錄下：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

確認：
- Health: `http://localhost:8000/health`
- Swagger: `http://localhost:8000/docs`

## 8. 啟動 api-gateway

在 `api-gateway` 目錄下：

```bash
npm install
npm run dev
```

確認：
- Health: `http://localhost:3001/health`
- Models: `http://localhost:3001/api/models`

## 9. 啟動 frontend

在 `frontend` 目錄下：

```bash
npm install
npm run dev
```

開啟：
- `http://localhost:3000`
- V2 入口：`http://localhost:3000/v2`

## 10. 正確啟動順序

1. 啟動 Ollama
2. 啟動 Docker 基礎服務：`docker compose up -d`
3. 套用 migrations：`npm --prefix api-gateway run migrate:deploy`
4. 啟動 `ai-service`
5. 啟動 `api-gateway`
6. 啟動 `frontend`

## 11. V2 相關說明

- V2 前端路徑：`/v2`
- Legacy 頁面：`/legacy`
- V2 語音流程：
  1. 瀏覽器用 Web Speech API 把語音轉文字
  2. 前端呼叫 `/api/v2/voice/intent`
  3. 後端解析 `destination / days / budget / preferences`

所以如果你問「語音模型是哪個模型」：
- 語音轉文字：不是 Ollama 模型，是瀏覽器 Web Speech API
- 意圖解析：目前不是 LLM，是 `ai-service` 的規則解析
- 對話模型：`gemma4:e4b`
- Embedding 模型：`nomic-embed-text`

## 12. 常見問題

### Q1. `/api/models` 或 `/api/chat` 打不通
- 先確認 `api-gateway` 有啟動
- 再確認 `NEXT_PUBLIC_API_BASE_URL=http://localhost:3001`

### Q2. V2 語音按鈕可按，但沒有辨識結果
- 先確認使用 Chrome
- 確認瀏覽器已允許麥克風權限
- 確認目前裝置支援 `SpeechRecognition` 或 `webkitSpeechRecognition`

### Q3. 推薦或規劃沒有結果
- 確認 migration 已包含 `014_v2_core_schema.sql`
- 確認 V2 schema 已建立
- 確認資料表內已有 V2 測試資料

### Q4. Ollama 有啟動，但推薦仍異常
- 確認 `OLLAMA_BASE_URL`
- 確認 `gemma4:e4b` 與 `nomic-embed-text` 都已 pull 完成

### Q5. Docker 指令找不到
- 表示 Docker 沒有在 PATH
- 可改用：

```powershell
& 'C:\Program Files\Docker\Docker\resources\bin\docker.exe' compose ps
```
