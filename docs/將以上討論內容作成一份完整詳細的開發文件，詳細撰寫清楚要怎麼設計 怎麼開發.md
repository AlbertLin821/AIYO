# 將以上討論內容作成一份完整詳細的開發文件，詳細撰寫清楚要怎麼設計 怎麼開發

這是一份針對「在 AI 旅遊規劃平台中，用對話精準擷取使用者偏好與過往經驗，並用來做 YouTube 影片推薦、片段推薦、安排行程」的完整開發文件。
你專案裡可以把它當作「技術設計書」來實作，所有技術細節都會用 Python + Ollama + FastAPI + PostgreSQL 示範，並避免儲存影片本體，只存 metadata 和時間戳。

***

## 一、系統目標與整體架構

### 1.1 系統目標

- 在與使用者的對話中，用 Ollama 模型自動擷取使用者偏好與過往旅遊經驗。
- 用這些資訊做：
    - YouTube 影片推薦（整支影片＋片段推薦）。
    - 旅遊行程規劃（天數、活動、交通、天氣、景點）。
- 只在資料庫中儲存影片相關資訊（title、author、url、video_id、start_time、summary），不存影片本身。
- 影片片段可直接點擊，跳轉到 YouTube 播放器的指定時間點。


### 1.2 系統架構（層級劃分）

- **前端**（React / TypeScript）
    - 顯示對話視窗、行程步驟、影片卡片、片段跳轉連結。
- **後端**（FastAPI）
    - 處理對話、呼叫 Ollama、呼叫工具（YouTube、天氣、交通、嵌入），並存資料到資料庫。
- **AI 核心**（Ollama + Tool Calling）
    - 用 `llama3.1` 或 `qwen2` 做對話、行程規劃、工具呼叫。
- **工具層**（Python modules）
    - 用 YouTube Data API、Whisper、Ollama embedding、cosine similarity 做影片推薦與片段搜尋。
- **資料庫**（PostgreSQL）
    - 存 user_profile、youtube_videos、video_segments。

這種架構在 AI 旅遊規劃系統裡非常常見，實作上穩定且可擴展。[^1][^2]

## 二、對話中精準擷取使用者偏好與過往經驗（User Profiling）

### 2.1 設計使用者偏好資料結構

首先在後端定義一個 `user_profile`，記錄使用者的偏好與過往旅遊經驗，不存影片，只存描述與標籤：

```python
user_profile = {
    "user_id": "xxx",
    "language": "zh_TW",
    "age_group": "20-30",
    "travel_types": ["情侶出游", "親自旅遊", "背包客"],
    "interests": ["夜市", "登山", "拍照打卡", "美食", "文化歷史"],
    "budget_level": "中等",
    "trip_duration_days": 3,
    "preferred_video_style": "vlog",
    "max_video_duration_min": 15,
    "must_have_subtitles": True,
    "past_trips": [
        {"location": "台南", "keywords": ["夜市", "小吃", "安平古堡"]},
        {"location": "花蓮", "keywords": ["太魯閣", "海"]},
    ],
    "negative_preferences": ["夜市人潮過多", "長時間走路"]
}
```

這結構可以直接映射到 PostgreSQL 表格，後端用 SQLAlchemy 存取。

### 2.2 用 Ollama 工具動態更新 `user_profile`

讓 Ollama 在對話中自動更新 `user_profile`，不需要靠問卷表單，而是用 `update_user_profile` 工具，讓 model 自行分析對話，然後呼叫這工具更新。

```json
tool_extract_preference = {
  "type": "function",
  "function": {
    "name": "update_user_profile",
    "description": "Extract user preferences and past travel experience from chat history and update user profile",
    "parameters": {
      "type": "object",
      "properties": {
        "language": {"type": "string"},
        "age_group": {"type": "string"},
        "travel_types": {"type": "array", "items": {"type": "string"}},
        "interests": {"type": "array", "items": {"type": "string"}},
        "budget_level": {"type": "string"},
        "trip_duration_days": {"type": "integer"},
        "preferred_video_style": {"type": "string"},
        "max_video_duration_min": {"type": "integer"},
        "must_have_subtitles": {"type": "boolean"},
        "past_trips": {
          "type": "array",
          "items": {
            "type": "object",
            "properties": {
              "location": {"type": "string"},
              "keywords": {"type": "array", "items": {"type": "string"}}
            }
          }
        },
        "negative_preferences": {"type": "array", "items": {"type": "string"}}
      },
      "required": ["interests"]
    }
  }
}
```

後端接收 `tool_calls` 後，用 SQLAlchemy 存到資料庫，更新使用者 profile，日後所有推薦都用這個 `user_profile`。[^2][^3]

## 三、YouTube 影片推薦與內容處理（只存 metadata）

### 3.1 只存影片 metadata，不存影片本體

你想的很對：只存影片的標題、作者、網址、時間戳，而不存影片本身。這正是 YouTube 推薦系統的常見做法，用 metadata + 外部 API 做內容索引。[^4][^5]

使用 `YouTube Data API` 抓 metadata：

```python
from googleapiclient.discovery import build

def get_youtube_metadata(video_id: str):
    youtube = build("youtube", "v3", developerKey=YOUTUBE_API_KEY)
    request = youtube.videos().list(
        part="snippet",
        id=video_id
    )
    response = request.execute()
    item = response["items"][^0]
    return {
        "video_id": video_id,
        "title": item["snippet"]["title"],
        "author": item["snippet"]["channelTitle"],
        "description": item["snippet"]["description"],
        "url": f"https://www.youtube.com/watch?v={video_id}",
        "duration_seconds": convert_duration_iso_to_seconds(item["contentDetails"]["duration"]) if "contentDetails" in item else None,
    }
```

資料庫 schema：

```sql
CREATE TABLE youtube_videos (
    video_id TEXT PRIMARY KEY,
    title TEXT,
    author TEXT,
    url TEXT,
    description TEXT,
    duration_seconds INTEGER
);
```

這部分在 YouTube 推薦研究中也常被用到，用 metadata 來做索引是標準實務，而不是直接存影片本體。[^5][^4]

### 3.2 用 Whisper + embeddings 做語意搜尋（片段時間戳）

要做「片段推薦」，你用的是 Whisper + embeddings 的語意搜索，用 cosin similarity 找到最相關的時間戳，這是 YouTube 時間戳搜尋的常見技術路線之一。[^6][^7]

流程：

1. 下載影片音訊（用 `yt-dlp` 或 `pytube`）。
2. 用 `Whisper` 轉為字幕，每個字幕有 `start`、`end`、`text`。
3. 用 `Ollama` 的 `nomic-embed-text` 做 embeddings，做 cosine similarity。
4. 用 `start` 時間做 YouTube 播放器跳轉。
```python
import whisper
import json
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

def download_and_transcribe(video_id: str):
    from pytube import YouTube
    yt = YouTube(f"https://www.youtube.com/watch?v={video_id}")
    audio_stream = yt.streams.filter(only_audio=True).first()
    audio_path = f"audio/{video_id}.mp3"
    audio_stream.download(output_path="audio/", filename=f"{video_id}.mp3")

    model = whisper.load_model("base")
    result = model.transcribe(audio_path)
    
    segments = []
    for segment in result["segments"]:
        segments.append({
            "start": segment["start"],
            "end": segment["end"],
            "text": segment["text"]
        })
    
    with open(f"transcripts/{video_id}.json", "w") as f:
        json.dump(segments, f, ensure_ascii=False)
```

然後用 Ollama embeddings 做語意搜尋：

```python
import ollama

def embed_text(text: str) -> np.ndarray:
    resp = ollama.embeddings(model="nomic-embed-text", prompt=text)
    return np.array(resp["embedding"])

def search_video_segment(video_id: str, query: str, max_results: int = 5) -> list[dict]:
    with open(f"transcripts/{video_id}.json") as f:
        segments = json.load(f)

    query_emb = embed_text(query)
    results = []

    for seg in segments:
        seg_emb = embed_text(seg["text"])
        score = cosine_similarity([query_emb], [seg_emb])[^0][^0]
        results.append({
            "start": seg["start"],
            "end": seg["end"],
            "text": seg["text"],
            "score": score
        })

    results.sort(key=lambda x: x["score"], reverse=True)
    return results[:max_results]
```

這方法在 `YT-Transcriber-QuerySearch`、`YouTube 用 Whisper 搜尋時間戳` 的專案裡都證明非常有效。[^8][^6]

## 四、影片推薦：挑出最符合使用者需求的 5 支影片

你要的是：模型先用 YouTube API 找一批影片，再用內容語意篩選，挑出 5 支最符合使用者需要的影片，而不是隨機給五支。

流程：

1. 用 YouTube API 搜尋 20～50 支影片。
2. 用 `user_profile` 和 `video metadata` 做 embeddings，用 cosine similarity 打分。
3. 排名前 5 的影片回傳給使用者。
```python

def rank_videos_by_preference(videos: list[dict], user_profile: dict) -> list[dict]:
    user_desc = " ".join([
        " ".join(user_profile["interests"]),
        " ".join(user_profile["travel_types"]),
        " ".join(user_profile["negative_preferences"])
    ])
    user_emb = embed_text(user_desc)

    ranked = []
    for v in videos:
        text = f"{v['title']} {v.get('description', '')}"
        vid_emb = embed_text(text)
        score = cosine_similarity([user_emb], [vid_emb])[^0][^0]

        if user_profile["preferred_video_style"] == "vlog" and "vlog" in v["title"].lower():
            score += 0.1

        if v.get("duration_seconds", 0) > user_profile.get("max_video_duration_min", 15) * 60:
            score -= 0.15

        ranked.append({
            "video": v,
            "score": score
        })

    ranked.sort(key=lambda x: x["score"], reverse=True)
    return [item["video"] for item in ranked[:5]]
```

這樣你就在 Ollama 工具中，用 `user_profile` 做推薦，而不是隨機挑影片。這在旅行推薦系統中被稱為「content-based filtering」，用內容與使用者偏好做語意匹配，而不是只用協同過濾。[^9][^2]

## 五、整合影片推薦到旅遊行程規劃（Agent Loop）

你要的是：在行程規劃中，自動插入影片推薦，並用片段時間戳跳轉，而不是分開做兩件事。

### 5.1 Ollama Agent Loop（整合工具與推薦）

你用的是 Ollama 的 tool calling，讓模型在行程規劃時，自動呼叫 `search_travel_videos`、`search_video_segment` 等工具，再把結果用 `role: "tool"` 回傳，形成一個 AI Agent Loop。這在 Ollama 官方文件中已被證明是穩定做法。[^10][^11]

```python
def travel_planning_agent(user_input: str, user_profile: dict):
    tools = [
        tool_extract_preference,
        tool_weather,
        tool_youtube,
        tool_segment_search,
        tool_web_search,
        tool_transport
    ]

    messages = [
        {"role": "system", "content": "你是專業 AI 旅遊規劃助手，擅長用工具取得即時資訊。"},
        {"role": "user", "content": user_input}
    ]

    while True:
        response = ollama.chat(
            model="llama3.1",
            messages=messages,
            tools=tools,
            stream=False
        )
        msg = response["message"]
        messages.append(msg)

        if "tool_calls" not in msg:
            break

        for tool_call in msg["tool_calls"]:
            name = tool_call["function"]["name"]
            args = tool_call["function"]["arguments"]

            if name == "update_user_profile":
                # 用 user_profile 更新，直接儲存到 DB
                messages.append({"role": "tool", "tool_name": name, "content": "Profile updated."})

            elif name == "search_travel_videos":
                videos = search_travel_videos(
                    location=args["location"],
                    activity_type=args["activity_type"],
                    max_results=30
                )
                filtered_videos = rank_videos_by_preference(videos, user_profile)
                # 用字串整理成影片清單，回給 Ollama
                result_str = "\n".join([
                    f"- {v['title']} ({v['url']})" for v in filtered_videos
                ])
                messages.append({
                    "role": "tool",
                    "tool_name": name,
                    "content": result_str
                })

            elif name == "search_video_segment":
                segments = search_video_segment(
                    video_id=args["video_id"],
                    query=args["query"],
                    max_results=5
                )
                result_str = "\n".join([
                    f"- {int(seg['start'])}s: {seg['text']}" for seg in segments
                ])
                messages.append({
                    "role": "tool",
                    "tool_name": name,
                    "content": result_str
                })

            # 其他工具類似 ...
    return response["message"]["content"]
```

這段在 `Ollama` + `MCP` 的 AI Travel Planner 專案中也有實作，用多工具串流程，讓 AI 自動做多步驟決策。[^12][^11]

### 5.2 將時間戳與影片摘要存入資料庫，讓前端可跳轉

你只希望存：

- `video_id`
- `start_time_seconds`
- `summary`
- `video_segments` 表格

```sql
CREATE TABLE video_segments (
    id SERIAL PRIMARY KEY,
    video_id TEXT REFERENCES youtube_videos(video_id),
    start_time_seconds INTEGER,
    summary TEXT,
    relevance_score NUMERIC
);
```

前端用 `video_id` + `start_time_seconds` 跳轉：

```bash
https://www.youtube.com/watch?v=VIDEO_ID&t=START_TIME_SECONDS
```

這樣你完全不需要存影片，只要用 `video_segments` 表格，就可以做時間戳跳轉，這在 YouTube 用 Whisper 搜尋時間戳的專案中也是相同做法。[^13][^6]

## 六、實作建議與專案結構（可直接抄進你的專案）

你可以用這個結構：

- `core/`：行程規劃、Ollama 代理、工具調用。
- `tools/`：`tools_youtube.py`、`tools_weather.py`、`tools_transport.py`。
- `embeddings/`：`transcript_processor.py`（用 Whisper + Ollama embedding）。
- `db/`：`models.py`（SQLAlchemy 用於 `user_profile`、`youtube_videos`、`video_segments`）。
- `api/`：FastAPI endpoints，`/chat`、`/plan-trip`、`/video_segment`。

前端用 React，用 `fetch` 呼叫 `/plan-trip`，用 `/video_segment` 獲取時間戳，用 `a` 連結或 `iframe` 跳轉到指定時間點。
