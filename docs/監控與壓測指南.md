# 監控與壓測指南

## 目標

- 監控 `api-gateway` 與 `ai-service` 服務健康度與延遲。
- 建立 p95 超過 500ms 的告警條件。
- 透過 Locust 驗證 10 位使用者併發下 E2E 延遲目標（< 2s）。

## Sentry

### Gateway

- 透過 `SENTRY_DSN` 啟用，程式位置：`api-gateway/src/index.js`。
- 建議環境變數：
  - `SENTRY_DSN=<your-dsn>`
  - `NODE_ENV=production`

### AI Service

- 透過 `SENTRY_DSN` 啟用，程式位置：`ai-service/app/main.py`。

## Prometheus 指標

### Gateway

- 指標端點：`GET /metrics`
- 主要指標：
  - `gateway_http_request_duration_ms`
  - 預設 Node process 指標

### AI Service

- 指標端點：`GET /metrics`
- 使用 `prometheus-fastapi-instrumentator` 自動輸出 FastAPI 指標。

## 告警建議

- 規則：`gateway_http_request_duration_ms` 的 p95 > 500ms，持續 5 分鐘告警。
- 另加：
  - 5xx 比例 > 2%
  - fallback 次數持續上升

## Locust 壓測

### 安裝

```bash
pip install locust
```

### 執行

```bash
locust -f benchmark/locustfile.py --host http://localhost:3001
```

### 建議情境

- Users: `10`
- Spawn rate: `2/s`
- Duration: `5-10 分鐘`
- 觀察：
  - `/api/chat` 平均值與 p95
  - `/api/user/memory` 成功率
  - Gateway 與 AI service 的 metrics 變化

## 驗收門檻

- E2E 平均延遲：< 2s
- `/api/chat` p95：接近或低於 2.5s
- 無大量 5xx 與無連續 fallback 異常峰值
